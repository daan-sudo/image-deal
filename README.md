# AI 图片筛选与评审工作台

> 一个面向设计团队和内容团队的桌面端图片工作台，帮助用户高效完成大批量图片的浏览、比较、筛选、整理与决策。

---

## 一、产品定位

### 1.1 目标用户

| 用户角色 | 核心场景 | 痛点 |
|---------|---------|------|
| **设计师** | 收到一批 AI 生成图、设计稿截图后需快速筛选出最优方案 | 逐张打开对比效率低，缺乏结构化评审工具 |
| **运营/市场人员** | 活动素材、参考图的汇总与决策 | 收集的图片散落各处，缺少统一筛选和标注平台 |
| **产品经理** | 竞品截图整理、UI 方案评审与反馈 | 无法高效组织素材并沉淀结论 |
| **自由创作者** | AI 出图后批量审图、整理灵感素材 | 传统图片查看器无法标记和分组 |

### 1.2 核心场景

1. **批量审图**：一次性导入数十甚至上百张图片，快速浏览并做出「保留 / 淘汰 / 待定」的判断
2. **对比评审**：将候选图片放在一起并排对比，辅助最终决策
3. **分类整理**：通过标签、分组、评分等方式组织图片，沉淀结论
4. **结果导出**：将筛选结果（目录/标注）导出，便于团队协作和归档

### 1.3 产品价值主张

**不止于「看图」，而是「做决策」。** 本工作台聚焦于从「大量原始素材」到「最终筛选结论」的全流程效率提升，是一个真正帮助用户做判断和决策的图片工作台。

### 2.2 核心功能清单

### 3.2 关键交互流程

#### 流程一：导入 → 快速筛选

```
拖入文件夹 → 自动扫描加载 → 网格视图浏览
  → 点击图片查看大图 → 键盘快捷键快速标记（A=通过, D=淘汰, S=待定）
  → 筛选出「通过」图片 → 导出到目标文件夹
```

#### 流程二：对比评审

```
在网格中多选候选图片 → 进入对比视图
  → 并排查看，同步缩放细节 → 添加评审备注
  → 确定最终方案 → 标记通过/淘汰
```

#### 流程三：分类整理

```
创建分组（如「风格A」「风格B」「最终方案」）
  → 浏览图片并拖入对应分组
  → 为关键图片打标签 → 导出评审报告
```

### 4.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **框架** | Electron + Vue 3 | 桌面应用壳 + 前端渲染 |
| **语言** | TypeScript | 类型安全 |
| **构建** | electron-vite | Electron 专用 Vite 构建工具 |
| **状态管理** | Pinia | Vue 3 官方推荐状态管理 |
| **路由** | Vue Router | 页面/视图切换 |
| **UI 组件** | 自研 + 少量第三方 | 追求清晰克制的视觉风格 |

| **文件访问** | Electron File API | 通过 Electron 主进程读取本地文件系统 |

### 4.4 纯前端方案说明

本项目 **不使用后端服务**，所有逻辑均在本地完成：

- **文件读取**：通过 Electron 主进程的 `fs` 模块读取本地文件系统
- **数据存储**：使用 IndexedDB（通过 Dexie.js 封装）在浏览器端持久化项目数据、标记、评分等
- **缩略图生成**：通过 Canvas API 在前端生成缩略图并以 DataURL 存储
- **导出功能**：通过 Electron 主进程将文件写入指定目录

## 六、设计理念

### 6.1 为什么这样设计

| 设计决策 | 理由 |
|---------|------|
| **三态标记而非简单删除** | 审图过程中常需反复权衡，「待定」状态允许用户推迟决策，减轻单次判断压力 |
| **并排对比视图** | 这是区别于普通图片查看器的核心功能。比较是决策的本质，单独看一张图往往无法做出判断 |
| **键盘快捷键驱动** | 审图经常是数十上百张的批量场景，鼠标点击效率太低，键盘快捷键可大幅提升吞吐量 |
| **左中右三栏布局** | 左侧负责导航筛选，中央负责浏览，右侧负责详情编辑，符合用户浏览-选择-操作的自然心智模型 |
| **纯前端无后端** | 图片审查是本地行为，无需网络传输；本地存储够用且更安全，避免引入不必要的架构复杂度 |
| **Electron 桌面端** | 需要直接访问本地文件系统，Web 应用受限于浏览器沙箱无法满足大批量文件操作的需求 |
| **IndexedDB 持久化** | 相比 localStorage 容量更大，适合存储大量图片元数据和缩略图数据 |

### 6.2 界面设计原则

- **清晰**：信息层次分明，核心操作醒目
- **克制**：不堆砌功能，每个功能都服务于「筛选决策」这一核心目标
- **直观**：交互符合直觉，降低学习成本
- **高效**：为高频操作提供快捷路径（键盘快捷键、批量操作）

## 九、值得解决与暂不解决的问题

### ✅ 值得优先解决

1. **高效浏览体验** — 大量图片加载的性能优化（虚拟滚动、懒加载缩略图）
2. **快速决策流程** — 键盘驱动的标记系统，减少鼠标操作
3. **有效对比能力** — 并排对比是区别于普通看图软件的关键差异点
4. **数据持久化** — 关闭应用后工作进度不丢失

### ❌ 暂不解决

1. **AI 自动标注/分类** — 虽然产品名带有「AI」，但 MVP 阶段关注人工效率而非自动化
2. **云端同步/多人协作** — 纯本地应用不涉及网络功能
3. **图片编辑** — 不是本产品的核心目标，专注于「筛选」而非「修图」
4. **视频素材支持** — 第一版聚焦图片场景
